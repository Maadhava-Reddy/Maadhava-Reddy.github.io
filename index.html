NetPulse — Multi-threaded TCP Chat Server

featured
C++17 · Linux · epoll · pthreads/std::thread · GNU Make

An event-driven chat server built to understand what actually happens between accept() and a message arriving on another client's screen — non-blocking I/O, readiness notification, and safe shared state under concurrency.

clients ──TCP──▶ epoll event loop (1 thread, O_NONBLOCK, drain to EAGAIN)
                        │ submit(task)
                worker thread pool (std::thread · mutex · condition_variable)
                        │ broadcast — fd snapshot, no lock held on send()
                shared client table (std::mutex)
  • Handles 200+ concurrent client connections in load tests via a Python stress harness.
  • Graceful lifecycle: SIGINT-driven shutdown, fd teardown, worker join; SIGPIPE ignored, MSG_NOSIGNAL on sends.
  • Make build system with release and AddressSanitizer/UBSan debug targets; races debugged with GDB and strace.
